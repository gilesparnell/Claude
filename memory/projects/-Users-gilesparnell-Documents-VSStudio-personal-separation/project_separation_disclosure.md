## Tool gotchas learned

- `modify_doc_text` requires character indices; no native find-and-replace. Use `import_to_google_doc` for rewrites (new doc, new ID)
- No "move file" or "trash file" tools — user does manually
- `create_drive_file` upload is sandboxed to `/Users/gilesparnell/.workspace-mcp/attachments/` — stage local files there first via `cp`, then upload, then clean up
- For PDFs in Drive: `get_drive_file_content` (pypdf text extraction)
- For LOCAL PDFs: Read tool extracts as images; Bash `mv` for renaming
- HTML-escape ampersands cause `&amp;` literal — use raw `&`
