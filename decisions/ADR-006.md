Build-time pre-rendering via @prerenderer/rollup-plugin rather than server-side rendering. The site is a Vite SPA with ~15 public pages — pre-rendering generates static HTML at build time that search engines can crawl, without requiring a Node.js server in production. Simpler infrastructure, no runtime SSR cost, and sufficient for the current page count. Revisit if page count exceeds ~50 dynamic routes.
