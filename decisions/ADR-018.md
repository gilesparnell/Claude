A review of Anthropic's internal prompt engineering course (Christian Ryan, May 2026) against the existing global CLAUDE.md revealed three structural gaps: (1) rules described desired outputs but showed no concrete examples — "rules without examples drift" as the course puts it; (2) no explicit rule required verifying file paths or function names before citing them, so stale memory could be presented as current fact; (3) critical rules (timezone, TDD, commit gate, verify-before-done) were stated once deep in a 580-line file, where they tend to be forgotten mid long session.

Six targeted edits addressed all three gaps without violating the 20-line growth brake. Added: a "Verify Before Citing" section (cite nothing from memory without grep/find verification), inline example entries for `decisions.md` and `handoff.md`, a good/bad CHANGELOG example, and a "Critical Reminders" block at the document end repeating the four most regression-prone rules. Removed: the redundant Timezone landmark section (rule already at the top), and tightened the "Relationship to existing conventions" block from 6 lines to 2. Net: +11 lines.

The Anthropic course also validated the existing skills-over-prose-rules pattern: procedural instructions belong in always-active skills (`tdd-first`, `verification-before-completion`) not inline CLAUDE.md rules, and that architecture is already in place. The `/think` flag for extended thinking on planning tasks was noted as underused and is a low-cost improvement available without any CLAUDE.md change.
