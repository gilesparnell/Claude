The card-grid-per-page portal stopped scaling: previously written docs were hard to find, there was no global search, and 14 of 23 skills were silently unpublished because generate-stats.js drops any skill lacking a `title:` field. Adopted a shared shell (docs/portal-shell.js + portal.css additions) that gives every page a persistent left sidebar, a Pagefind-backed ⌘K command palette, and a scannable list view rendered from a single unified docs-index.json (built by scripts/build-docs-index.js). Skills and decisions render as dense lists; walkthroughs and templates keep curated cards — lists for inventory, cards for visual curation. The index generator + Pagefind are wired into the build-portal.yml CI. Direction was researched via last30days + web: sidebar-plus-breadcrumbs and search-first are the dominant scalable documentation patterns (Mintlify, Starlight, Stripe docs). All 13 missing skills were backfilled with portal frontmatter so the library is fully published.
