On 2026-04-21 Claude was asked to adopt seven `/insights` report recommendations for the global Claude setup. Instead of checking for semantic overlap with existing `~/.claude/CLAUDE.md` sections and always-active skills, Claude built a plan to add all seven. Three were already covered (the `verification-before-completion` skill, Shipping Discipline, Compute Efficiency Defaults, and Project Status Catch-Up). Giles had to prompt Claude to pause and think before the noise landed.

The fix is a pre-flight triage rule now in global CLAUDE.md: for any "adopt X" / "add these recommendations" request, Claude must produce an explicit **accept / reject / defer** triage against current state before writing any plan. The reject column must exist — if it's empty, re-check for overlap. Paired with a CLAUDE.md growth brake: any proposal adding >20 lines requires a paired proposal for what to remove or migrate to a skill. Skills are preferred over prose rules because they don't load every turn.
