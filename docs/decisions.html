All Infrastructure Architecture Tooling Process
ADR-001
Rules in config, never in code
All business rules — keywords, queue mappings, timing offsets, state detection — are stored in an external config file or database table, never hardcoded in the engine. This means rule changes never require a deployment.
accepted FSCA automation March 2026
ADR-002
Railway + Vercel over single-platform hosting
Backend (Node.js + PostgreSQL) on Railway, frontend (React dashboard) on Vercel. Keeps concerns separated, both have generous free tiers, and each can be scaled independently as engagements grow.
accepted infrastructure hosting March 2026
ADR-003
Git submodule for shared toolkit, not private npm
The shared Parnell Systems toolkit is consumed via git submodule rather than a private npm registry. Zero cost, works with any git host, no registry setup needed. Graduate to private npm when 3+ client repos are consuming it and the API is stable.
accepted tooling architecture March 2026
ADR-004
Vite + React SPA over Next.js
Marketing site and future customer dashboards built as a Vite + React SPA with React Router, not Next.js. GHL is the backend — we're building a frontend that talks to GHL's API, not a full-stack app. Pre-rendering covers SEO for the ~15–30 public pages. Migrate to Next.js only if content marketing scales beyond pre-rendering limits (50+ dynamic SEO pages).
accepted frontend infrastructure March 2026
ADR-005
GHL native voice agents as current delivery platform
All customer voice agents delivered via Go High Level's native voice functionality. LiveKit and Vapi are future-state options for custom deployments requiring more control, but GHL native is current for all sales, demos, and client deployments. Plans and marketing materials must not reference LiveKit/Vapi as current capabilities.
accepted voice-ai ghl platform March 2026
ADR-006
@prerenderer/rollup-plugin for SEO over SSR
Build-time pre-rendering rather than server-side rendering. The site is a Vite SPA with ~15 public pages — pre-rendering generates static HTML at build time that search engines can crawl, without requiring a Node.js server in production. Revisit if page count exceeds ~50 dynamic routes.
accepted seo frontend March 2026
ADR-007
useSEO custom hook over react-helmet-async
A zero-dependency custom useSEO hook for managing page meta tags instead of react-helmet-async. The pre-renderer captures the DOM after React renders, so meta tags set via direct DOM manipulation are picked up in static HTML output. Avoids a runtime dependency for something the build process handles. Sets title, description, canonical URL, OG tags, and JSON-LD structured data.
accepted seo frontend dependencies March 2026
ADR-008
Organisation + SoftwareApplication JSON-LD, not LocalBusiness
Structured data uses Organisation + SoftwareApplication + FAQPage schema. Parnell Systems is a remote SaaS business — LocalBusiness would be factually incorrect and could trigger Google penalties. SoftwareApplication schema supports pricing, ratings, and feature data. FAQPage schema delivers highest SERP real estate on the pricing page.
accepted seo structured-data March 2026
ADR-009
Pricing tiers: $399/$599/$799 with $2,000 setup and tiered annual discounts
Three flat-rate tiers: Starter ($399/mo), Business ($599/mo), Professional ($799/mo). Annual billing at 1 month free. Setup fee: $2,000 for all tiers. Annual discounts on setup: Starter full, Business 50% off ($1,000), Professional waived. Monthly billing always pays full setup. Existing customers get 90-day loyalty bridge.
accepted pricing stripe revenue March 2026
ADR-010
Two-stage reactivation model for AU compliance
Database reactivation campaigns use a two-stage permission-pass model to comply with AU Spam Act 2003 and Privacy Act 1988. Stage 1: single permission-pass SMS. Stage 2: only explicit opt-ins receive the full campaign. All numbers washed against DNCR before any voice outreach. SMS Sender ID Register compliance required from 1 July 2026.
accepted compliance sms australia March 2026
ADR-011
AllConvos demo separation strategy
Voice AI demos must migrate off AllConvos to parnellsystems.com before full product separation. Demo pages on parnellsystems.com use GHL voice widgets directly, with each niche having its own widget ID, phone number, and audio sample. The AllConvos codebase must not be modified — it belongs to Jesse.
accepted allconvos demos migration March 2026
ADR-012
No brand name commitment — use Parnell Systems
Do not hardcode any product brand name (previously "ZingCall" was considered). All customer-facing materials use "Parnell Systems" with a "voice" sub-brand badge until a final product name is decided. Code, config, URLs, and marketing copy must avoid baking in a name that may change.
accepted branding naming March 2026
ADR-013
Stripe Payment Links over embedded checkout
Pricing page CTAs link to Stripe Payment Links rather than embedded checkout or a custom payment flow. No backend required, Stripe handles the full checkout UX including tax collection. Each tier and billing interval gets its own Payment Link URL. Trade-off: less control over checkout experience, appropriate for current scale.
accepted stripe payments March 2026
ADR-014
90-day pro-rata money-back guarantee, excludes setup fee
New customers get a 90-day pro-rata money-back guarantee on subscription. The $2,000 setup fee is non-refundable — it covers real delivery work. Reduces buyer friction for a high-ticket SaaS without exposing setup costs to refund risk. Worst case Business Annual: cancel at day 89, refund ~$4,510, keep ~$2,480 for 3 months of service.
accepted pricing customer-trust refunds March 2026
ADR-015
Execute GTM plan week-by-week as independent work items
The 8-week GTM plan is executed one week at a time, each as an independent /ce:work scope. Keeps each session focused, avoids burning credits planning work that's weeks away, and allows the plan to adapt based on prior-week results. Giles completes manual tasks (Stripe, Pixel IDs, etc.) before Claude runs code tasks with real values.
accepted workflow planning execution March 2026
ADR-016
Triage-first for adoption of any recommendation set
For any "adopt X" / "add these recommendations" request, Claude must produce an explicit accept/reject/defer triage against current state before writing any plan. The reject column must exist — if empty, re-check for overlap. Paired with a CLAUDE.md growth brake: any proposal adding >20 lines requires a paired proposal for what to remove or migrate to a skill.
accepted workflow claude discipline April 2026
ADR-017
Ship /ship and /resume as skills, not commands
Skills compose with the skill-forced-eval hook on UserPromptSubmit, show up in the live skill list, and reference other skills by declared triggers. /ship composes with the existing verification-before-completion skill for its merge-verification step rather than re-implementing gh pr view — composition over duplication. /resume automates the Project Status Catch-Up rule with an explicit stop-and-ask gate.
accepted claude skills workflow April 2026
ADR-018
Apply prompt engineering best practices to global CLAUDE.md
Anthropic prompt engineering course review revealed three gaps: rules without examples drift; no verify-before-cite rule; critical rules stated once in 580 lines get forgotten mid-session. Six targeted edits fixed all three: added Verify Before Citing, inline examples for decisions/handoff/CHANGELOG, and a Critical Reminders block. Removed two redundant sections. Net: +11 lines, under the 20-line growth brake.
accepted claude claude-code prompt-engineering May 2026
