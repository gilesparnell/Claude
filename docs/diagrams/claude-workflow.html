Waklthrough — Workflow
Claude Chat vs
Claude Code
Two tools, one workflow. This chat is for planning, designing, and generating. Claude Code is for building, executing, and pushing.

This chat (claude.ai) is your thinking and planning environment. Claude Code (in your terminal) is your execution environment. The repo is the handoff point. This walkthrough shows how they work together.

Key principles:
  – Never copy-paste between the two — the repo is the handoff
  – The tech spec is the contract — Claude Code reads it and builds to it
  – The portal updates itself — push a skill, it appears automatically
  – CLAUDE.md is the memory — lessons learned travel with the codebase
